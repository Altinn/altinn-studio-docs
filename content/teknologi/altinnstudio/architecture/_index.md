## How is the Technical Architecture documented?
The [non functional and functional requirements](/teknologi/altinnstudio/architecture/requirements/) have given input to which capabilities 
for both devops and runtime that is needed in the solutions. It also have given input which type of components that is needed to implement the capabilties.
The [Architecture Principles](/teknologi/altinnstudio/architecture/principles/) and [Architecture Patterns](/teknologi/altinnstudio/architecture/patterns/) has been
decided based on requirements and other goals with the new solutions. The Principles and patterns has been deciding factors how the components should implement the capabilities needed.
